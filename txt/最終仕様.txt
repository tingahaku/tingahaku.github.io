ポケグラフ 最終仕様

このファイルの役割
- このファイルは、現時点の正式仕様を記録するための基準書とする。
- 仕様変更・UI変更・計算変更・文言変更を行った場合は、その都度このファイルを更新する。
- 今後の新名や作業者が「今どうなっているか」を把握するための参照元にする。
- 実装と仕様がずれた場合は、原則として「現在の実装に合わせてこのファイルを更新する」か、「実装を仕様に合わせて修正する」かを明示して揃える。
- 未確定の案や思いつきはここに書かず、確定した内容のみを書く。

更新ルール
- 変更後は、変更点だけでなく変更後の最終状態が読んで分かるように書く。
- 「昔はこうだった」は原則書かず、今の仕様だけを書く。
- 数値、色、文言、挙動のような変更されやすいものは、なるべく具体的に書く。
- 仕様変更時は、関連する項目も一緒に見直して矛盾をなくす。

アプリ概要
- サービス名は「ポケグラフ」。
- グラフ・耐性・技範囲・適正を使ってポケモンの特徴を可視化する、検索導線トップページと個別静的ページで構成されたアプリである。
- 初期表示ポケモンはガブリアス。
- 画面幅に余裕がある環境では、アプリ本体の左側余白に「注目のポケモン」サイドパネルを表示できる。
- 現在の注目表示は仮実装で、登録済みポケモンを50音順に並べ、件数不足ぶんは繰り返して100件表示する。
- 注目のポケモン一覧には、各行の左端に1〜100の順位番号を表示する。
- 注目のポケモンは、PC 幅では本文の左外側に付属するサイドパネルとして表示する。
- スマホ幅では注目のポケモンを本文の下に下部パネルとして表示する。
- PC 幅ではサイドパネルの上端をプロフィールの上端に合わせ、`sticky` でスクロール追従する。
- 本文側の通常レイアウトは広げず、サイドパネルのために本体配置を拡張しない。
- スマホ幅でも本文の各項目配置はPC版から大きく組み替えず、縦長のまま表示する。
- プロフィール名と注目のポケモン名は、表示幅が足りない場合は改行せず文字サイズを縮めて収める。

データ構成
- 初回ロードでは `data/PokemonIndex.json` のみを読み込む。
- ポケモン詳細は `data/pokemon/{key}.json` を個別に遅延読み込みする。
- `Ability.json` と `Move.json` は、最初にポケモン詳細が必要になった時点でまとめて読み込む。
- `scripts/Build-PokemonData.ps1` は `Pokemon.json` から `PokemonIndex.json` と個別ポケモン JSON を再生成するためのスクリプトである。
- `scripts/Build-StaticPages.ps1` は GitHub Pages 配信用に `docs/` 配下へ静的ページを生成する。
- ポケモンの内部処理には `key` を使い、`No` は表示専用とする。
- 画像参照には `imageKey` を使う。
- 正規URLは `/pokemon/{key}/` とする。
- `docs/index.html` は検索導線用トップページ、`docs/pokemon/{key}/index.html` は各ポケモンの個別静的ページとする。
- 個別静的ページでは HTML 自体がポケモンごとに独立して存在し、主要本文を初期HTMLとして埋め込む。
- 主要本文には、基本情報、弱点、能力、適正、起点作成、覚える技、技範囲、ひとくちメモを含める。
- 読み込み後は同じクライアントJSが動作し、検索、ツールチップ、切り替えなどの動的UIを補助する。
- 初回表示時は URL またはページ埋め込みの `key` を読んで該当ポケモンを表示する。
- ページタイトルは `ポケモン名の種族値・特性・適正・技範囲 | ポケグラフ` の形式で更新する。
- `meta description` は表示中ポケモンに応じて更新する。
- `scripts/Build-StaticPages.ps1` は `SiteUrl` を与えた場合、`canonical`、`og:url`、`sitemap.xml`、`robots.txt` も生成できる。
- GitHub Pages のサブディレクトリ配信でも崩れないよう、各静的ページは base path を埋め込み、CSS / JS / data / assets を相対基準で解決する。

検索仕様
- 検索対象は `PokemonIndex.json` に含まれるポケモン名。
- 正規化は以下を行う。
- 全角半角統一（NFKC）
- ひらがなをカタカナへ変換
- 空白除去
- 長音記号、ハイフン、中黒系記号の吸収
- 大文字小文字の統一
- 検索候補は部分一致で最大5件表示する。
- 完全一致した候補も検索候補内に表示する。
- Enter または検索ボタン押下で完全一致検索を行う。
- 完全一致した場合はそのポケモンの個別URLへ遷移する。
- 完全一致しない場合は検索欄下に「一致する名称がありませんでした」と表示する。
- 検索文字列が空の状態で検索した場合はガブリアスの個別URLへ遷移する。
- 検索確定後は検索欄の文字列を空にする。

検索 UI 仕様
- IME 変換中は再描画によって入力を壊さない。
- 検索候補は検索欄の直下に表示する。
- 候補行にはポケモン名とポケモンアイコンを表示する。
- 完全一致時は検索欄の左寄りから、対応ポケモンが検索欄の後ろから顔を出す演出を表示する。
- 顔出し演出は完全一致時のみ表示する。
- 顔出し演出はクリック可能で、クリックするとそのポケモンの個別URLへ遷移する。
- 顔出し演出はホバー時に少しだけ下がる。
- 顔出し演出は透明化で消えるのではなく、下へ潜ってから消える。
- 検索候補外クリックで候補を閉じる。

プロフィール表示
- 上部に図鑑番号、名前、ポケモン画像、タイプ、特性を表示する。
- 特性は最大3枠表示し、不足分は `-` を表示する。
- ひとくちメモがある場合のみメモ欄を表示する。

タイプアイコン仕様
- タイプ画像は `assets/types/*.png` を使う。
- タイプ画像ファイル名は英語版タイプ名の正式スペルに合わせる。
- 対応は以下の通り。
- normal
- fire
- water
- electric
- grass
- ice
- fighting
- poison
- ground
- flying
- psychic
- bug
- rock
- ghost
- dragon
- dark
- steel
- fairy

能力グラフ仕様
- 能力欄にはレーダーチャートを表示する。
- グラフ凡例は以下とする。
- 種族値
- しんかのきせき
- 特性発動時
- 系列色は以下とする。
- 種族値: 青
- しんかのきせき: 薄紫
- 特性発動時: オレンジ
- グラフ上部の白い領域内にボタンを2つ置く。
- `HPを統合`
- `種族値のみ`
- `HPを統合` はオンオフ式の切り替えだが、ボタン文言自体に ON/OFF は書かない。
- `HPを統合` オフ時は通常6軸表示。
- 軸は HP / 攻撃 / 防御 / 素早さ / 特防 / 特攻 の順で表示する。
- `HPを統合` オン時は5軸表示。
- 軸は 攻撃 / 物理耐久 / 素早さ / 特殊耐久 / 特攻 の順で表示する。
- 物理耐久は `sqrt(HP × 防御)` を整数切り捨てした値を使う。
- 特殊耐久は `sqrt(HP × 特防)` を整数切り捨てした値を使う。
- しんかのきせき補正や特性補正込み系列も、補正後能力値を使って同様に物理耐久・特殊耐久を算出する。
- `HPを統合` オン時のグラフ最大値も 200 に統一する。
- `種族値のみ` オン時は、しんかのきせき・特性発動時の系列を非表示にする。
- グラフの軸ラベル文字と種族値数値は拡大済み。
- 軸ラベルの下には種族値を表示する。
- しんかのきせきや特性で値が変わる軸については、種族値の下に補正後数値を追加表示する。
- 補正後数値は凡例と同じ色で表示する。
- 補正後数値は変化した軸にのみ表示する。

能力値・計算の丸め仕様
- 能力値系の表示は整数切り捨てを基本とする。
- しんかのきせき補正後の防御・特防は整数切り捨て。
- 特性補正込み能力値も整数切り捨て。
- 適正・指数・耐性点・耐久点などの補助計算は、小数第1位切り捨てベースで扱う。
- 掛け算が続く計算は、なるべく原作寄りに「段階ごと切り捨て」で処理する。
- 適正値もその段階切り捨ての積み上げ結果を表示する。

適正欄仕様
- 適正欄には以下を表示する。
- 物理アタッカー適正
- 特殊アタッカー適正
- 物理スイーパー補正
- 特殊スイーパー補正
- 物理受け適正
- 特殊受け適正
- 各項目は数値、バー、ツールチップボタンを持つ。
- 適正バーの最大値は固定で 400 とする。
- ツールチップには計算要素と倍率を改行付きで表示する。
- 各適正ツールチップの最上段には、その適正の意味を説明する一文を表示する。
- 適正の基礎計算は「ステータス要素は足し算、補正要素は掛け算」を基本とする。
- 物理アタッカー適正: `(攻撃 + 総合耐久点) × 特性補正 × シナジー補正`
- 特殊アタッカー適正: `(特攻 + 総合耐久点) × 特性補正 × シナジー補正`
- 物理スイーパー補正: `(攻撃 + 素早さ) × 特性補正 × シナジー補正`
- 特殊スイーパー補正: `(特攻 + 素早さ) × 特性補正 × シナジー補正`
- 物理受け適正: `(HP + 防御) × タイプ耐性点 × 特性補正 × シナジー補正`
- 特殊受け適正: `(HP + 特防) × タイプ耐性点 × 特性補正 × シナジー補正`

起点作成欄仕様
- 適正欄の下に数値ではない「起点作成」欄を置く。
- 旧来の「起点作成適正点」「起点作成適正バー」は廃止し、数値評価は行わない。
- 項目は以下の6つとする。
- 設置技
- 状態異常
- バトン
- 対面操作
- 妨害
- サポート
- 各項目は押せないボタンのようなチップで表示する。
- チップは小さめに表示し、3列×2段で並べる。
- 覚える技の中に対応タグを持つ技が1つでもあれば、その項目を明るく表示する。
- 該当技がなければ灰色表示する。
- 技データに旧来の数値的な起点作成点は持たせず、タグだけで判定する。
- チップにカーソルを乗せると、対応する技名一覧を上側のツールチップで表示する。
- 起点作成見出しの右隣にも `i` マークを置き、`戦局を有利にする適正` と表示する。

弱点欄仕様
- 左上プロフィール下に通常の弱点一覧を表示する。
- 4倍、2倍、0.5倍、0.25倍、無効をタイプアイコンで列挙する。

技範囲ヒートマップ仕様
- 技範囲は全タイプ組み合わせのヒートマップで表示する。
- 各マスはタイプ組み合わせに対する通り倍率を表示する。
- 上見出しと左見出しにはタイプ名を表示する。
- 上見出しは黒地白字で、上にタイプカラー帯を表示する。
- 左見出しは黒地白字で、左にタイプカラー帯を表示する。
- タイプカラー帯は参考にした自作タイプ見出し画像の色味へ寄せてある。
- 重複もカウントボタンを持つ。
- サブ範囲を強調ボタンを持つ。
- 重複表示オフ時は同タイプ順列も数える。
- 重複表示オン時は重複マスを灰色表示する。
- サブ範囲強調オン時は、自タイプ由来より通りが良くなる組み合わせを強調表示する。

技範囲ホバーツールチップ仕様
- 各マスにホバーすると、対象タイプ組み合わせと倍率を2行で表示する。
- 例:
- ほのお・ひこう
- 4倍
- 単タイプ扱いのマスは `ノーマル・ノーマル` ではなく `ノーマル` のように1タイプだけ表示する。
- ツールチップは改行以外では折り返さない。
- 右端でもなるべく切れないよう、ヒートマップ外へはみ出して表示できるようにしている。

技範囲サマリー仕様
- ヒートマップ右にサマリー欄を置く。
- スマホ表示時のみ、ヒートマップを上、縮小したサマリー欄を下に置く。
- スマホ表示時のヒートマップは横スクロール可能とし、表が横にはみ出して暴走しないようにする。
- 上部に以下2ボタンを置く。
- 重複もカウント
- サブ範囲を強調
- その下に以下5ブロックを置く。
- 4倍
- 2倍
- 0.5倍
- 0.25倍
- 無効
- 各ブロックは中央寄せで表示する。
- 倍率ラベルは大きめ表示。
- 数字も大きめ表示。
- バーは数字の下に表示する。
- バー色はヒートマップ色の濃い版に寄せる。
- 4倍: 赤系
- 2倍: オレンジ系
- 0.5倍: 水色系
- 0.25倍: 青系
- 無効: 青灰系

技範囲の選択仕様
- 技範囲下に4枠の攻撃技タイプ選択欄を置く。
- ポケモン選択直後のみ、初期値として自タイプを先頭から自動セットする。
- ユーザーが手で未選択に変更した場合は、その状態を維持し、勝手に元タイプへ戻さない。
- すべて未選択にした場合も、その状態を維持する。
- 新しいポケモンを検索・選択したときは、そのポケモンに対して再度初期化を行う。
- 各選択欄の下には、選択タイプに対応する候補技プレビューを表示する。
- 候補技は物理上位2件・特殊上位2件を優先し、足りなければ他候補で補う。

覚える技欄仕様
- 覚える技欄は `物理` `特殊` `変化` のタブで切り替える。
- 初期タブは、特攻が攻撃より高ければ特殊、そうでなければ物理。
- 技カードには以下を表示する。
- 技名
- 技タグ（ある場合のみ。技名の下段に小さく表示）
- タイプアイコン
- 威力
- 命中
- 説明
- 分類アイコンは表示しないが、物理 / 特殊 / 変化 の分類概念自体はタブ切り替えに使う。
- STAB が乗る攻撃技は、威力の横に補正後威力を括弧付きで表示する。
- 技タグは複数付与可能とする。
- 現在のテストタグは以下。
- じならし: 妨害
- ステルスロック: 設置技

エラー表示
- Ability.json に存在しない特性、Move.json に存在しない技はエラー欄に表示する。
- そのデータは表示対象から除外する。
- 初期化失敗時は初期化エラーパネルを表示する。

見た目・UI方針
- タイプアイコンは自作画像へ差し替え済みであることを前提にする。
- ヒートマップ中央マスには色を使うが、見出し部分は黒地白字ベースで整理する。
- 検索演出やツールチップなど、遊びのある要素は維持する。
- ただし、可読性を損なう場合は見た目より実用性を優先する。

TypeChart.js、TypeCalculator.js、MoveRange.js、Resistance.js作成指示書

ポケグラフ用に、TypeChart.js、TypeCalculator.js、MoveRange.js、Resistance.js を作成してください。

【前提】
- ES Modules（イーエスモジュールズ：JavaScriptの分割読み込み方式）で分割すること
- 外部ライブラリは禁止
- 毎行に日本語コメントを付けること
- まず動作優先で、できるだけシンプルに書くこと
- この指示に書かれていない独自仕様の追加は禁止
- 不明点は勝手に補完せず、安全な最小実装にすること

【確定仕様】
- タイプ相性データは TypeChart.js に固定データとして持たせる
- タイプ相性は「攻撃側タイプ -> 防御側タイプ -> 倍率」の形で管理する
- TypeChart.js に書かれていない組み合わせは 1 倍として扱う
- 無効は 0 倍として扱う
- 複合タイプは掛け算で処理する
- 例えば ほのお技 vs くさ・はがね なら 2 × 2 = 4 倍
- 例えば でんき技 vs みず・じめん なら 2 × 0 = 0 倍
- 全タイプ一覧も TypeChart.js から export する

【作成してほしいファイル】

1. TypeChart.js
- 次の定数を export すること
  - TYPE_CHART
  - ALL_TYPES
- TYPE_CHART の中身は、こちらが渡す完成データをそのまま使うこと
- 勝手にタイプ名を英語化しないこと
- 勝手に並び順を変えないこと

2. TypeCalculator.js
- 攻撃タイプ1個、防御タイプ配列、TYPE_CHART を受け取り、最終倍率を返す関数を作ること
- 関数名は getTypeMultiplier とすること
- 引数例：
  - attackType: "ほのお"
  - defenseTypes: ["くさ", "はがね"]
  - typeChart: TYPE_CHART
- 戻り値は number（ナンバー：数値）
- 書かれていない相性は 1 倍として計算すること

3. Resistance.js
- 防御側タイプ配列を受け取り、全18タイプから受ける倍率を計算して分類する関数を作ること
- 関数名は calculateResistance とすること
- 引数例：
  - defenseTypes: ["ドラゴン", "じめん"]
  - typeChart: TYPE_CHART
  - allTypes: ALL_TYPES
- 戻り値は次の形の object（オブジェクト：まとまりデータ）にすること
  {
    weak4: [],
    weak2: [],
    resist05: [],
    resist025: [],
    immune: []
  }
- 1倍のタイプは配列に入れないこと

4. MoveRange.js
- 技タイプ配列を受け取り、全タイプ組み合わせに対する通りの良さを計算する関数を作ること
- 未選択は計算対象外とすること
- 複数攻撃タイプがある場合は、最も高い倍率を採用すること
- 単タイプは「同じタイプどうしの組み合わせ」として扱うこと
  例：
  - ほのお × ほのお は単タイプほのおとして扱う
- 返すデータは、ヒートマップ表示に使いやすい配列形式にすること
- 例：
  [
    {
      type1: "ほのお",
      type2: "ひこう",
      multiplier: 2
    }
  ]
- 関数名は calculateMoveRange とすること

【MoveRange.js の仕様詳細】
- 入力は最大4個の技タイプ配列を想定する
- 例：
  ["じめん", "ドラゴン", null, null]
- null や空文字は無視すること
- 各防御側タイプ組み合わせに対して、入力された技タイプそれぞれの倍率を計算し、その中の最大値を採用すること
- 4倍弱点数、2倍弱点数、0.5倍耐性数、0.25倍耐性数、無効数も数えられるようにすること
- 可能なら、ヒートマップ本体データと集計データをまとめて返すこと
- 返り値の例：
  {
    cells: [
      { type1: "ノーマル", type2: "ノーマル", multiplier: 1 }
    ],
    summary: {
      weak4: 0,
      weak2: 0,
      resist05: 0,
      resist025: 0,
      immune: 0
    }
  }

【防御側タイプ組み合わせの扱い】
- 全18タイプ × 全18タイプでループしてよい
- ただし、同じ組み合わせの順番違いは重複扱いしないこと
  例：
  - ほのお・ひこう と ひこう・ほのお は同一として1つだけ作る
- そのため、二重ループでは内側ループを外側と同じ添字から始めること
- 同じタイプどうしなら単タイプとして扱うこと

【テスト用コードも作成してほしい】
- 各ファイルの最後に、コメント付きで簡単な使用例を書くこと
- ただし本番 export を壊さない形にすること
- 例として次の結果が確認できるようにすること
  1. getTypeMultiplier("こおり", ["ドラゴン", "じめん"], TYPE_CHART) が 4
  2. getTypeMultiplier("でんき", ["みず", "じめん"], TYPE_CHART) が 0
  3. calculateResistance(["ドラゴン", "じめん"], TYPE_CHART, ALL_TYPES) で こおり が weak4 に入る
  4. calculateResistance(["ゴースト"], TYPE_CHART, ALL_TYPES) で ノーマル と かくとう が immune に入る
  5. calculateMoveRange(["じめん", "ドラゴン", null, null], TYPE_CHART, ALL_TYPES) が配列または object として正常に返る
